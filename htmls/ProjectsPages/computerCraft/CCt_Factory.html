This is still an in-dev multi-script, multi-computer behaviour.

But, it is to automate production of materials in the Create mod, using recursive tasks to define what items need to be made, what tasks must wait, etc etc...

No download yet. still in development.

How it functions:

To make things simple, there is a Main computer, a Task computer, and a Storage computer, all shortened to M T S, all communicating via wired-modems.

The M-computer gets data about items in storage from the S-computer, then the M-computer decides which items need to be produced from their amounts in storage.

From there, the T-computer gets a 'task', a ticket of what needs to be produced, with how to make it, and how many inputs are necessary.

Upon the completion of that task, the T-computer puts the item in the S-computer's storage, and tells the M-computer it's finished!

The M-computer will then check what items are in storage again, and starts over.
